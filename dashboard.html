Targeted Deauth — aireplay-ng -0
Sends 802.11 deauth frames. Most effective for forcing handshake capture. Use count=0 for continuous.
Deauth/Disassoc Amok — mdk4 d
Amok mode targets all APs/clients in range. Extremely disruptive.
Auth DoS — mdk4 a
Floods AP with fake auth frames — exhausts association table. Requires BSSID.
Beacon Flood — mdk4 b obsolete
Fills scan lists with fake SSIDs. Less effective on modern devices.
WIDS/WIPS Confusion — mdk4 w obsolete
Fake management frames to confuse intrusion detection systems.
Michael Shutdown — mdk4 m TKIP only
Exploits TKIP MIC failure — forces 60s AP shutdown. Only works on WPA-TKIP, no effect on WPA2-CCMP.

PMKID Hash Cracking
After PMKID capture completes, the hash file is saved to /tmp/fufu-sec/pmkid_hash.txt. Crack it with hashcat mode 22000 (WPA-PBKDF2-PMKID+EAPOL) or aircrack-ng.
Step 1 — Inspect Hash File
Step 2a — hashcat (Recommended — GPU accelerated)
Step 2b — aircrack-ng (CPU, slower)
aircrack-ng can crack PMKID hashes only if the hash file was also saved as a .cap (pcapng). Use the Convert tab on the Handshake page first, or crack the raw 22000 hash with hashcat.

Attack Explanation
ARP Replay (-3)
Captures and replays ARP requests to generate IVs. Fastest if a client is active. Needs ~50k IVs.
Chopchop (-4)
Decrypts last byte of a packet without knowing the key. Slower but works without active clients.
Fragmentation (-5)
Gets keystream from a single packet. Very effective when no clients are present on the network.
Caffe-latte (-6)
Attacks isolated clients using their cached WEP keys. Useful in public hotspots.
